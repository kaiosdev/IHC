# Etapa 3: Benchmarking

O objetivo desta etapa é estudar soluções similares para compreender como problemas semelhantes aos identificados no briefing são abordados por outras ferramentas.

Com base no briefing com o Professor Victor Celso, o problema central é a falta de uma ferramenta **específica para a gestão de projetos acadêmicos da UFAM**, o que o força a usar ferramentas genéricas que exigem muito **retrabalho de adaptação**.

O benchmarking foi focado em analisar essas ferramentas genéricas (mencionadas pelo cliente) e os sistemas oficiais (que não cumprem a função de acompanhamento).

### a) Avaliação dos itens a serem comparados

Com base na demanda, os itens definidos para comparação foram:

* **Gestão de Projetos:** Capacidade de organizar e acompanhar a evolução de tarefas.
* **Especificidade Acadêmica (UFAM):** Se a ferramenta possui modelos prontos para projetos (PIBIC, PACE, Pibex, etc.).
* **Retrabalho de Adaptação:** O esforço necessário para o usuário configurar a ferramenta para a "linguagem UFAM" (criar tags, planilhas, cabeçalhos, etc.).
* **Centralização de Editais:** Capacidade de agregar informações e links externos (como os sites das Pró-Reitorias).
* **Integração (E-campus):** Potencial de integração com outros sistemas de banco de dados.
* **Portabilidade:** Qualidade da experiência de uso em múltiplos dispositivos (Desktop e Mobile).

### b) Definição dos similares-alvo

Os produtos, organizações ou serviços selecionados como referência foram:

* **Sistema 1: Trello**
    * *Descrição: Ferramenta de gestão visual de projetos baseada em quadros Kanban. Mencionada pelo cliente.*
* **Sistema 2: Notion**
    * *Descrição: Ferramenta "tudo-em-um" que combina anotações, bancos de dados e gestão de projetos. Mencionada pelo cliente.*
* **Sistema 3: Google Sheets / Excel**
    * *Descrição: Ferramentas de planilhas. Mencionadas pelo cliente como a alternativa manual que exige "fazer uma planilha toda formatada".*
* **Sistema 4: E-campus / SEI**
    * *Descrição: Sistemas oficiais da UFAM. Analisados para confirmar a lacuna de funcionalidade de "acompanhamento" identificada pelo cliente.*
